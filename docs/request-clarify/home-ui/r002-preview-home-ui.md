# r002-preview-home-ui 需求澄清

日期：2026-05-03

## 需求结论

根据 `preview.html`，本次需求是将当前 Web 首页调整为深色本地优先笔记工作台。界面核心从当前的浅色双栏和顶部输入框，改为左侧统计与筛选导航、右侧笔记 feed、底部悬浮 composer 的布局。

## 预览稿关键信息

| 模块 | 预览稿表现 | 当前实现差异 |
| --- | --- | --- |
| 整体主题 | 深色背景、深色面板、浅蓝强调色 | 当前为浅米色背景、白色卡片 |
| 布局 | 桌面端固定宽度两栏：左侧 300px，右侧 feed 760px，中间 64px 间距 | 当前为响应式两栏，左侧 260px，输入框在 feed 顶部 |
| 品牌区 | `MeNote` + `LOCAL` badge | 当前为 `Zembra` + `卡片笔记` |
| 左侧统计 | 笔记数、标签数、连续/累计天数三项统计 | 当前没有统计区 |
| 活跃热力图 | 12 列小方格热力图 + 月份标签 | 当前没有热力图 |
| Fields 导航 | `@` 前缀、名称、数量、当前选中态 | 当前没有 Field 筛选 |
| Tags 导航 | `#` 前缀、名称、数量 | 当前只有从当前 notes 推导出的 tag 按钮 |
| 搜索 | 右上角胶囊搜索框，提示“搜索笔记、Field、Tag”，展示 `⌘+K` | 当前搜索在左侧，只有笔记内容搜索 |
| 笔记卡片 | 深色圆角卡片、meta 行、右上角菜单、正文、标签 pill、展开/引用信息 | 当前只显示正文和标签 |
| 输入框 | 底部固定悬浮 composer，带工具栏、保存目标提示和发送按钮 | 当前输入框在列表顶部 |
| 辅助入口 | 右下角圆形 `?` 按钮 | 当前没有 |

## 功能范围建议

本次推荐先实现静态视觉与现有数据能力的对齐：

| 功能 | 范围 |
| --- | --- |
| 笔记 feed | 本轮先做视觉占位，不接现有 notes 列表能力 |
| 标签筛选 | 本轮先做视觉占位，不实现真实筛选 |
| 搜索 | 入口移动到右上角搜索框，本轮保留输入状态占位 |
| 创建笔记 | 底部悬浮 composer 本轮先保留输入和插入格式，不提交到 notes 后端 |
| 标签解析 | 本轮暂不进入真实 notes 数据流 |
| Fields | 先按预览稿做 UI 占位或从后端 `/fields` 接入，需用户确认 |
| 统计和热力图 | 先用 notes/tags 派生基础数值，热力图可先做静态视觉占位，需用户确认 |

## 待确认问题

1. 品牌名保留 `Zembra`，不使用预览稿里的 `MeNote`。
2. Fields 本次接入后端 `/fields`，并在侧栏展示真实 field 列表。
3. 统计区第三项先使用占位，不实现累计或连续天数计算。
4. 笔记卡片里的“置顶”“展开”“被 N 条笔记引用”和右上角菜单，本次只做视觉，不实现真实交互。
5. 底部 composer 工具栏里的 `#`、`@`、格式按钮需要支持插入格式功能，发送按钮保持可用。

## 已确认范围

| 功能 | 结论 |
| --- | --- |
| 品牌 | 保留 `Zembra`，可保留 `LOCAL` badge 表示本地优先 |
| Fields | 接入 `/fields`，侧栏展示真实字段；筛选行为按 API 能力和现有 notes DTO 限制设计 |
| 统计第三项 | 先占位 |
| 卡片扩展功能 | 只做视觉表现 |
| Composer 工具栏 | 实现插入 `#`、`@` 和基础 Markdown 格式片段 |
| Notes 数据 | 已确认后端提供 `POST /notes/recent`，本轮接入最近 50 条笔记作为首页 feed |

## 暂不默认纳入

- 不新增第三方 UI 套件。
- 不引入富文本编辑器。
- 不实现全局 Command Palette，除非用户确认 `⌘+K` 要可用。
- 不新增 UI 自动化测试，除非用户明确允许。
- 不接当前 `/notes` 作为首页 feed 数据源；首页 notes 使用 `POST /notes/recent`。
