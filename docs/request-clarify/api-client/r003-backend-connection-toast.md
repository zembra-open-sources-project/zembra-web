# Backend 连接失败 Toast 提示需求澄清

日期：2026-05-10

## 需求结论

当 Web 前端请求 backend 时，如果浏览器无法连接到 backend，需要显示 toast 提示，并且 toast 必须在 5 秒后自动消失，不能常驻。

## 范围

| 项目 | 结论 |
| --- | --- |
| 触发条件 | `fetch` 网络层失败，例如 backend 未启动、连接被拒绝、DNS 或 CORS 导致请求没有拿到 HTTP 响应。 |
| 不触发条件 | backend 返回 HTTP 4xx/5xx 时继续走现有 `ApiError` 逻辑，不作为“连接不上 backend”。 |
| 展示位置 | Web 应用全局展示，避免每个页面重复处理。 |
| 展示时长 | 每次提示显示 5 秒，到期自动消失。 |
| 依赖变化 | 不新增第三方 toast 依赖。 |

## 验收标准

- backend 无法连接时，页面出现明确的连接失败提示。
- toast 5 秒后自动消失。
- 同一时间不出现多个重复的 backend 连接失败 toast。
- 现有 API 错误解析和页面加载逻辑不被破坏。
