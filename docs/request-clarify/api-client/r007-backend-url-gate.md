# R007 Backend URL Gate 需求澄清

日期：2026-05-15

## 需求结论

用户访问 WebUI 时，需要先输入后端 URL。前端检查该 URL 可达后，保存到浏览器本地配置，并进入当前首页笔记页。后续访问时先读取已保存 URL 并检查可达性；如果不可达，则回到 URL 输入页并显示错误提示。

## 范围

| 项目 | 结论 |
| --- | --- |
| 配置内容 | 仅保存后端 API Base URL |
| 权限验证 | 当前不做账号、密码、token 或权限校验 |
| 配置位置 | 浏览器 `localStorage`，满足 WebUI 本地配置需求 |
| 可达性检查 | 对输入 URL 发起轻量 HTTP 请求，网络失败或不可访问视为不可达 |
| 成功后页面 | 进入现有首页，即笔记页 |
| 失败后页面 | 停留或返回 URL 输入页，并显示错误提示 |

## 验收标准

- 首次访问没有已保存 URL 时显示后端 URL 输入页。
- 输入可达 URL 后保存配置并渲染笔记首页。
- 输入不可达 URL 时不进入首页，并显示错误提示。
- 已保存 URL 在后续访问时会先检查可达性。
- 已保存 URL 不可达时清除当前会话门禁并回到输入页。
