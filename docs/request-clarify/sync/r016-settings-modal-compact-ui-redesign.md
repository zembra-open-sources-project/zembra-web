# r016-settings-modal-compact-ui-redesign 需求澄清

日期：2026-05-30

## 背景

用户在 2026-05-30 提供当前 Settings 模态（Supabase 配置区块）的真实截图，明确反馈“非常丑陋”。经过多轮交互澄清，确认问题根源并非配色，而是**布局结构和控件呈现方式**：

- 当前使用单个浅色大容器 + 内部表格式网格（FieldLabel / SettingRow 的 150px 固定列 + 底边框），形成强烈的“传统配置面板 / 数据库管理表格”视觉，过于陈旧、拥挤、缺乏现代感。
- 与 App 其他部分（NoteCard、白卡片 + 极细边框语言）不一致。
- 输入控件 affordance 弱、Toggle 过小、底部操作按钮视觉权重不足。

用户明确要求在**不违背现有整体配色风格**的前提下进行重构。

## 需求理解

本次需求是对 SettingsModal 中 Supabase section 的**纯视觉与布局重构**，目标是获得“紧凑、专业、现代”的设置面板体验，同时严格保持功能不变。

核心设计决策（经过用户多轮确认）：
- 采用**方案二**：将配置拆分为两个语义分组的独立紧凑白色卡片（“连接信息” + “同步设置”）。
- 整体模态宽度显著收窄。
- 输入框必须**右对齐**，**完全去掉边框和背景**，仅用**一条细下划线**作为可编辑的视觉表达。
- 追求**极致紧凑**（用户反复强调“紧凑、紧凑、紧凑，绝对禁止再增大列行高”），在保证中文标签完整显示、不截断的前提下，把所有间距、padding、行高压到最低。
- 两张卡片之间保留 **4px** 间隙。
- 同步设置区块内的两个 value（间隔秒数、启用开关）同样必须右对齐。
- 100% 复用现有 CSS 变量和配色系统，绝不引入新颜色。

## 已确认结论

| 项目               | 最终确认结论                                                                 |
|--------------------|-----------------------------------------------------------------------------|
| 布局形态           | 两个独立白色紧凑卡片（连接信息 + 同步设置），而非单一表格容器                |
| 模态最大宽度       | max-w-[480px]（显著窄于当前 640px）                                          |
| 输入框表现         | 仅使用 1px 底部细线（border-b），无任何左右/上边框、无背景；focus 时切换为 accent 强色线 |
| 对齐方式           | 所有输入 value（包括同步设置的数字和开关）均右对齐                          |
| 卡片间距           | 两张卡片之间保留 4px 间隙                                                   |
| 紧凑程度           | 行垂直 padding 压至 py-[5px]~py-[6px]；卡片内 padding 控制在 px-3 py-2 左右；Label 列宽约 110px（保证“同步间隔秒数”不换行） |
| 配色约束           | 严格复用 --color-surface、--color-border、--color-border-strong、--color-accent、--color-text-* 等全部现有变量，禁止新 hue |
| 功能范围           | 仅视觉与布局重构，保存、测试、启用开关的交互逻辑与 API 调用完全不变         |
| 中文显示           | 所有标签必须完整显示，不允许截断或强制换行                                  |
| 按钮风格           | 保持现有 icon + 文字的低调风格，仅做必要紧凑化处理                           |

## 范围

- 重构 `SettingsModal` 的整体宽度与内边距。
- 重构 `SupabaseSettingsSection` 的布局结构：
  - 移除原有的单一 `bg-[var(--color-surface-muted)]` 表格容器。
  - 拆分为两个独立的紧凑卡片容器。
  - 重新设计 FieldLabel / SettingRow（或新建紧凑版组件）以支持“右对齐 + 仅下划线”的输入呈现。
- 对输入框、密码框、数字输入、Toggle、ActionButton 进行视觉紧凑化处理（保持功能）。
- 保证暗色模式下 CSS 变量自动适配（无需额外暗色专项代码）。
- 更新相关测试中的快照或视觉断言（如有）。

## 不做范围

- 不修改任何业务逻辑（保存流程、测试连接、enabled 切换、interval 校验等）。
- 不改动后端 API 契约或前端 API Client。
- 不新增/删除配置字段。
- 不改动 i18n 文案内容。
- 不引入任何新颜色、新组件库、新依赖。
- 不做完整的多 section 设置系统扩展（本次仅针对当前 Supabase section 的视觉重构）。
- 不单独为暗色模式编写特殊样式（完全依赖现有变量）。
- 本次不处理其他潜在设置项（若未来出现）。

## 验收标准

1. 视觉上明显比 r015 实现紧凑、现代、专业，无“旧后台表格”感觉。
2. 模态整体宽度显著收窄，内容更聚焦。
3. 两个白色卡片（连接信息 / 同步设置）清晰分组，之间保持 4px 间隙。
4. 所有输入 value 均右对齐，仅靠下方 1px 细线表达可编辑状态，focus 时线条使用 accent 强色。
5. 中文标签（尤其是“同步间隔秒数”）在目标宽度下完整显示，无截断、无强制换行。
6. 行高与卡片内边距在可读前提下达到用户要求的“极致紧凑”水平。
7. 完全复用现有配色系统，浅色/暗色模式下视觉一致且自然。
8. Test / Save 按钮位置、交互与之前一致，仅视觉更贴合紧凑整体。
9. 现有功能（加载配置、保存、测试、实时切换 enabled）零退化。
10. 无新增控制台错误或可访问性问题。
