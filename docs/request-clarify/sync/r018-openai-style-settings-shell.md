# r018-openai-style-settings-shell 需求澄清

日期：2026-05-28

## 背景

用户希望将本项目设置页重构为接近 ChatGPT 网页设置页的样式。用户提供的参考图是 OpenAI 设置页中的 Security 面板，核心参考对象是其设置弹窗的空间模型和视觉语法，而不是 Security 页面中的具体账号安全功能。

本次澄清先固定 UI 布局边界，再进行功能澄清。后续功能澄清、设计文档和实现阶段禁止扭曲或修改本节确认的 UI 布局设计。

## UI 布局已确认结论

本次设置页采用 **OpenAI 风格单 surface 设置弹窗**：

- 整个设置页只有一个可见 modal surface。
- modal surface 内部划分为左侧分类导航区和右侧内容区。
- 左侧 sidebar 不得做成独立卡片、独立边框容器或嵌套面板。
- 右侧 content 不得做成独立卡片、独立边框容器或嵌套面板。
- 左右区域之间优先通过空间、对齐和内容层级区分，不使用显眼竖线或框中框。
- 关闭按钮属于左侧区域顶部的控制位，与左侧分类导航处在同一个 sidebar 布局流中。
- 左侧分类选中态使用浅色行背景或 pill，高亮的是 nav item，不是一个外层卡片。
- 右侧内容使用标题、水平分割线、设置行、右侧真实控件建立秩序。
- 设置项行使用 OpenAI 设置页式的平面列表表达，不使用 r016 的“双紧凑卡片”结构。
- URL、Secret key、Interval seconds 等本项目真实配置项仍然使用输入框，不改成箭头二级页。
- Enable sync 继续使用 switch。
- Test / Save 继续作为当前分类内容区内的真实动作按钮。

## 与既有方案的关系

r016 的设置页视觉方向是“窄 modal + 两个紧凑卡片 + 极致紧凑输入行”。本次 r018 明确切换到 OpenAI 风格设置中心：

| 项目 | r016 旧方向 | r018 新方向 |
| --- | --- | --- |
| 整体形态 | 窄单栏 modal | 宽双栏 modal |
| 内容组织 | 两个紧凑卡片 | 单 surface 内的平面设置行 |
| 视觉边界 | 卡片分组 | 标题、留白、分割线、行对齐 |
| 分类能力 | 不做完整多 section 设置系统 | 设置壳支持注册真实分类 |
| 输入控件 | 紧凑下划线输入 | 右侧真实输入框，样式贴合设置行 |

后续实现以 r018 为准，不再沿用 r016 的卡片布局。

## 功能澄清前置边界

功能澄清只能在已确认 UI 布局内展开：

- 可以澄清分类注册方式、当前分类名称、内容组件接入方式。
- 可以澄清当前 Supabase/Sync 设置项是否拆分成行、分组或动作区。
- 不得因为功能数量少而移除左侧导航。
- 不得新增 mock 分类或空分类来填充视觉。
- 不得把真实输入框改成二级箭头样式。
- 不得重新引入卡片、面板、外框等嵌套视觉结构。

## 功能范围初步理解

本次目标包含两层：

1. 建立一个可复用设置 shell，支持通过配置注册真实设置分类。
2. 将当前真实存在的 Supabase/Sync 设置作为首个分类接入该 shell。

## 功能已确认结论

| 项目 | 结论 |
| --- | --- |
| 当前分类名称 | 使用 `Sync`。 |
| 分类注册方式 | 使用 settings 模块内静态注册表，不引入 React context/provider 动态注册。 |
| 窄屏导航 | 左侧分类导航在窄屏下变为顶部横向分类栏，避免挤压右侧真实输入控件。 |
| 分类数据来源 | 只注册真实存在的分类，不新增空分类，不 mock 内容。 |
| 首个分类内容 | 当前 Supabase 同步配置作为 `Sync` 分类内容接入。 |

当前真实设置功能保持不变：

- 读取同步配置。
- 编辑 Supabase URL。
- 编辑 Secret key。
- 编辑 Interval seconds。
- 切换 Enable sync。
- 测试连接。
- 保存配置。
